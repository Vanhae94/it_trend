덕파

Opus 4.7 토큰 부담과 Codex의 기능 추격 속, Claude Code와 Codex는 우열이 아니라 용도로 갈린다

Opus 4.7의 토크나이저 변경으로 토큰 소모가 늘고 Claude Code에 연이은 사고가 겹치면서, GPT-5.5를 얹은 Codex가 빈틈을 파고들었습니다. 성능·한도·컨텍스트·기업 철학까지 뜯어본 결과 어느 한쪽의 압도적 우위는 없고, 작업 성격에 따라 도구를 나눠 쓰는 것이 결론입니다.

핵심 포인트

  • Opus 4.7 새 토크나이저는 외부 측정 기준 1.20~1.47배 더 많은 토큰을 쓰는 반면, GPT-5.5는 Codex 작업에서 약 40% 토큰 효율을 발표
  • AMD 스텔라 로렌조의 실측(6,852세션·234,760회 도구 호출)에서 Opus 4.7 이후 사고 깊이 중앙값이 67% 하락
  • SWE-bench Pro(64.3% 대 57.7%)·SWE-bench Verified(87.6% 대 74.9%)에서는 Opus 계열이 앞섬
  • Claude Code는 1M 컨텍스트를 추가 요금 없이 정식 지원, Codex는 400K이며 1M은 opt-in 단계
  • Anthropic은 '함께 가는 동료(companion)', OpenAI는 '위임받는 작업자(delegated worker)'라는 도구 철학 차이가 UX 설계에 그대로 반영

왜 지금

코딩 에이전트 시장의 무게중심이 Claude Code 독주에서 Codex와의 양강 구도로 옮겨간 흐름을 정량 데이터로 포착한 글로, 관측 4주 내내 인기 1위를 지키며 이 논쟁이 여전히 최상단임을 보여줍니다.

실무 시사점

장기·다중 파일 리팩터링이나 UI 마감이 잦은 팀은 Claude Code를, 명확한 스펙 기반 일상 코딩과 토큰 효율이 중요한 팀은 Codex를 우선 검토하고, 여유가 있다면 두 도구를 작업 유형별로 병행하는 전략이 현실적입니다.

실무 적용 포인트

  • 장기 리팩터링·1M 컨텍스트가 필요한 작업은 Claude Code, 스펙이 명확한 일상 코딩·토큰 절약이 중요하면 Codex로 작업을 나눠 배분해본다
  • 팀 내 코딩 에이전트 토큰 소모량을 주기적으로 측정해, 모델 업데이트 이후 체감 성능 변화를 데이터로 확인하는 습관을 들인다

가중치·대시보드가 아쉬웠던 유료 루틴 앱을, 원래 쓰던 Obsidian에 Git·Templater·Dataview + Claude Code(JavaScript)로 붙여 개인용 도구로 직접 만든 사례입니다. 가장 큰 난관인 기존 데이터 이전은 앱 통계 '스크린샷'을 생성형 AI에 읽혀 옮겼고, 결국 구독을 해지했습니다. '만드는 건 쉬워졌으니 무엇을 왜 만들지가 더 중요해졌다'는 결론을 담았습니다.

핵심 포인트

  • 개인용 도구의 기준은 '내가 매일 부담 없이 쓰는가' 하나뿐이라, 완성도보다 빠른 구현이 자연스러움(바이브 코딩의 강점)
  • Obsidian을 데이터 기반으로, Git(기기 동기화)·Templater(반복입력)·Dataview(조회)로 뼈대를 세우고, 대시보드·시각화 등 손 많이 가는 부분은 Claude Code가 JS로 구현
  • 가중치 루틴 체크(운동 1시간≠물 한 잔)·월/연간 히트맵 대시보드·루틴 메모 + 메모 기반으로 Claude에게 '내 생활 패턴'을 질문
  • 핵심 난관은 구현이 아니라 '기존 데이터 마이그레이션' — 손 입력을 포기하고 앱 통계 화면 스크린샷을 AI에 읽혀 규칙 설명 후 변환(잘못 넣은 달을 AI가 먼저 걸러내기까지)
  • 생성형 AI는 코드 작성만이 아니라 '반정형 데이터 마이그레이션'(화면엔 정리됐지만 손으로 옮기긴 귀찮은 정보)에서 큰 효율

3803(40대 비개발자)·3774(디자이너 앱 출시)와 함께, 바이브 코딩이 '개인이 직접 도구를 만들어 유료 서비스를 대체'하는 단계로 확산되는 이번 주 흐름을 대표합니다.

실무 시사점

매일 반복하는 불편·기존 도구의 아쉬움을 작은 개인용 도구로 직접 만들어보되, '무엇을 왜 만들지'와 '계속 쓰게 만들 구조'에 더 집중하는 것이 실질적입니다. 데이터 이전 같은 반정형 작업은 스크린샷+AI로 위임하면 갈아타기 성공률이 높아집니다.

  • 화면엔 정리됐지만 손으로 옮기긴 귀찮은 데이터는 스크린샷+규칙 설명으로 생성형 AI에 마이그레이션을 위임한다

핵심 포인트

  • 자율 에이전트는 권한 위계 최상단 — '여기까진 알아서'라고 정하면 자는 동안에도 24시간 무인 실행(가장 강력하고 가장 위험)
  • OpenClaw: 메신저에 봇으로 상시 상주(채널 50+), SOUL.md로 성격 고정, MS도 채택 — 셸·파일·메일 권한이 넓어 그대로 공격면
  • Hermes: 성공 절차를 스킬로 저장·재사용하는 '학습 루프'로 쓸수록 손에 맞춰짐(2026-05-10 OpenRouter 토큰 1위)
